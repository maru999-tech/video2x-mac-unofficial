# video2x-mac-unofficial

> ⚠️ **非公式・独立プロジェクト — 本家 Video2X ではありません。**
> 本家 [Video2X](https://github.com/k4yt3x/video2x)（k4yt3x 氏）とは**無関係・非公認**で、
> **コードも一切共有していません**。本家と**同じ下回りエンジン**（Real-ESRGAN / RIFE の
> `ncnn-vulkan`）を叩く、macOS向けの独立したシェルスクリプトです。名前の "video2x" は
> 「同じ2機能をやる」という意味だけです。本家が必要な方は上のリンクへ。

Video2X *相当*のローカル動画処理ツール（macOS / Apple Silicon 用）。
機能は Video2X と同じ **2つだけ**：

1. **超解像アップスケール** — Real-ESRGAN（`realesr-animevideov3`）で解像度を**倍率**で拡大
2. **フレーム補間** — RIFE（`rife-v4.6`）でフレームレートを**倍率**で増やし動きを滑らかに

解像度のリサイズ・アスペクト比変更・パディング等の追加加工は一切しません。
アップスケールで決まった解像度、補間で決まった fps が、そのまま出力に反映されます。

---

## 1. 基本の使い方

```bash
~/video2x/video2x.sh -i 入力ファイル [オプション]
```

出力は **入力と同じフォルダ**に `元ファイル名_v2x.mp4` という名前で作られます。

### 最短の例

```bash
# アップスケール×2 ＋ 補間×2（どちらも既定値）
~/video2x/video2x.sh -i movie.mp4
```

---

## 2. オプション一覧

| オプション | 内容 | 既定値 |
|---|---|---|
| `-i FILE` | 入力動画ファイル（**必須**） | なし |
| `-s N` | アップスケール倍率（`2` / `3` / `4` のみ） | `2` |
| `-f N` | 補間倍率（2以上の整数） | `2` |
| `-m MODEL` | アップスケールのモデル（`anime-video` / `anime` / `photo`） | `anime` |
| `-x N` | 出力の長辺の上限px（`0`=無制限） | `3840`（4K） |
| `-u on\|off` | アップスケールのオン/オフ | `on` |
| `-r on\|off` | フレーム補間のオン/オフ | `on` |
| `-h` | ヘルプ表示 | — |

### モデル（`-m`）の選び方

| 値 | 実体モデル | 特徴 | 向いている素材 |
|---|---|---|---|
| `anime`（既定） | `realesrgan-x4plus-anime` | 線・髪がくっきり。ディテール保持が良い | アニメ絵・AIイラスト動画（Dreamina等） |
| `photo` | `realesrgan-x4plus` | 実写寄りで質感多め。平面に細かいザラつきが出ることも | 実写・写実的な映像 |
| `anime-video` | `realesr-animevideov3` | 柔らかく滑らか（**水彩風に溶けやすい**） | ノイズの多い低画質アニメ動画の整え直し |

> **`-s` が 2/3/4 限定の理由**：これらのモデルが対応する倍率がこの範囲だからです（Video2X でも同じ）。
> `anime` / `photo` は **×4 専用**モデルのため、`-s 2` や `-s 3` を指定すると「×4 で処理 → 指定倍率へ高品質縮小（lanczos）」して合わせます。これは Video2X が固定倍率モデルで任意倍率を出すときと同じ方式です。

> **`-f` の意味**：元が 24fps で `-f 2` なら 48fps、30fps で `-f 2` なら 60fps。内部的には「元フレーム数 × 倍率」のフレーム数を RIFE で生成します。尺（再生時間）は変わりません。

## 3. 使用例

```bash
# ① アップスケール×2 ＋ 補間×2（フル処理）
~/video2x/video2x.sh -i anime.mp4 -s 2 -f 2

# ② アップスケールだけ（×4）、補間しない
~/video2x/video2x.sh -i anime.mp4 -s 4 -u on -r off

# ③ 補間だけ（×2）、アップスケールしない … 30fps→60fps 化など
~/video2x/video2x.sh -i clip.mp4 -f 2 -u off -r on

# ④ アップスケール×3 ＋ 補間×4
~/video2x/video2x.sh -i clip.mp4 -s 3 -f 4

# ⑤ モデルを実写寄りに変更（質感重視）
~/video2x/video2x.sh -i clip.mp4 -s 2 -f 2 -m photo

# ⑥ 旧来の柔らかいモデルに戻す
~/video2x/video2x.sh -i clip.mp4 -s 2 -f 2 -m anime-video

# ⑦ 解像度の上限を変える（長辺5120pxまで許可）
~/video2x/video2x.sh -i clip.mp4 -s 2 -f 2 -x 5120

# ⑧ 上限を外す（青天井・倍率どおり）
~/video2x/video2x.sh -i clip.mp4 -s 2 -f 2 -x 0
```

### 解像度の上限（`-x`）と「大きく作って小さく保存」

`元解像度 × 倍率` が上限（長辺px）を超えると、**モデルのネイティブ倍率で大きく生成 → 上限サイズへlanczos縮小**します（＝スーパーサンプリング）。これで:
- 出力ファイルが青天井に巨大化しない（既定は4K=長辺3840px）
- 大きく作ってから縮小するので、直接そのサイズを作るより**エッジがクリーンで締まった画**になる
- アスペクト比は保持、上限以下ならそのまま

> 例: 2560×1440 を `-s 2`（=5120×2880）にしても、既定では4Kに収まるよう **3840×2160** で保存。`-x 5120` にすれば 5120×2880 のまま、`-x 0` で無制限。

### 倍率の効果（早見表）

| 入力 | コマンド | 出力 |
|---|---|---|
| 640×480 / 24fps | `-s 2 -f 2` | 1280×960 / 48fps |
| 640×480 / 30fps | `-s 3 -u on -r off` | 1920×1440 / 30fps（fps変化なし） |
| 1280×720 / 24fps | `-f 2 -u off -r on` | 1280×720 / 48fps（解像度変化なし） |

## 4. 処理の流れ（内部パイプライン）

```
入力動画
  │
  ├─[1] 連番PNGに分解 ＋ 音声を別途抽出
  │
  ├─[2] Real-ESRGAN で各フレームを ×s アップスケール   （-u on のとき）
  │
  ├─[3] RIFE で ×f フレーム補間（元枚数×f 枚を生成）    （-r on のとき）
  │
  ├─[4] FFmpeg で再結合（fps×f で符号化）＋ 元の音声を戻す
  │
  └─[5] 中間ファイル（連番PNG）を自動削除
        → 元ファイル名_v2x.mp4
```

- 進捗はすべて日本語でターミナルに表示されます。
- 中間ファイルは macOS の一時フォルダに作られ、**処理後に自動削除**されます（途中でエラー終了しても消えます）。
- 音声は可能なら**無劣化コピー**、ダメなら AAC で再エンコードして戻します。音声トラックが無い動画もそのまま処理できます。
- 出力動画は H.264（`libx264` / `yuv420p` / `crf 16`）。一般的なプレイヤー・編集ソフトでそのまま再生・読み込みできます。

---

## 5. 出力の仕様

| 項目 | 内容 |
|---|---|
| コンテナ | MP4 |
| 映像コーデック | H.264（libx264, crf 16, preset slow） |
| ピクセル形式 | yuv420p |
| 解像度 | 入力解像度 × アップスケール倍率（補間OFFなら入力のまま） |
| フレームレート | 入力fps × 補間倍率（補間OFFなら入力のまま） |
| 音声 | 入力の音声をそのまま（無劣化コピー優先） |
| 尺 | 入力と同じ |

### 場所を変えたいとき（環境変数で上書き可能）

```bash
export VIDEO2X_TOOLS="$HOME/別の場所/tools"     # tools フォルダごと指定
export REALESRGAN_BIN="/path/to/realesrgan-ncnn-vulkan"
export RIFE_BIN="/path/to/rife-ncnn-vulkan"
export RIFE_MODEL="/path/to/rife-v4.6"
```

## 7. 困ったとき（トラブルシュート）

| 症状 | 対処 |
|---|---|
| `realesrgan が見つかりません` / `rife が見つかりません` | パスを確認。移動した場合は §6 の環境変数で指定。 |
| 「開発元を検証できない」等で起動しない | 検疫属性が残っている可能性。`xattr -dr com.apple.quarantine ~/video2x/tools` を実行。 |
| `アップスケール倍率(-s)は 2/3/4…` のエラー | `-s` は 2/3/4 のみ。それ以外は不可。 |
| `補間倍率(-f)は2以上…` のエラー | `-f` は 2 以上の整数で指定。 |
| 処理がとても遅い／重い | フレーム数×解像度に比例。長尺・高解像度・高倍率ほど時間がかかります。まず短いクリップで試してください。 |
| メモリ不足で落ちる | 4K へのアップスケール等は VRAM/メモリを多く使います。倍率を下げるか、入力を短く分割。 |
| 出力に音が無い | 入力に音声トラックが無い場合は仕様通り無音になります（`ffprobe 入力.mp4` で確認可）。 |
| 画面が真っ暗・本編が映らない・音楽だけ長く流れる | 入力が**可変フレームレート(VFR)**のときの症状。本ツールは「実フレーム数÷実再生時間」で出力fpsを決め、映像長＝音声長になるよう対策済み。古い版を使っている場合は最新の `video2x.sh` に置き換えてください。 |
| 画質が「水彩風に溶けた」感じ・線がぼやける | 既定が変わる前の `realesr-animevideov3` の特徴。`-m anime`（くっきり）や `-m photo`（質感重視）を試してください。元から高解像度・高画質な素材ほど効果的。 |

---

## 8. 注意・制約

- **対応倍率**：アップスケールは 2/3/4 のみ、補間は整数倍のみ。
- **追加加工なし**：特定解像度への合わせ込み・トリミング・色調整などは一切行いません（Video2X に無い処理は入れない方針）。
- フレーム補間は元映像に無い中間フレームを AI が生成するため、激しい動き・場面転換では稀に不自然なフレームが出ることがあります（RIFE 一般の特性）。
